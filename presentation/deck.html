01
The problem

Diligence is run by a committee of experts.
The committee is where deals slow down.

Buying one asset means title, zoning, environmental, structural, insurance and financial review. Those experts work in sequence over email and PDFs, and an associate assembles their findings by hand before the deal can close.

Slow & serial
3–6 wks

to clear diligence on a mid-market asset, most of it lost to hand-offs between specialists.

Costly to get wrong
$100M+

a single missed easement, zoning conflict or remediation liability can move an IRR below water.

No audit trail
Opaque

the rationale for an approval lives across inboxes — difficult to replay or defend to an IC.

Our read

The committee can run in minutes if every expert shares one context, the conflicts between them surface on their own, and each finding and number is recorded as it happens.

Problem
02
The solution

A standing committee of agents that
coordinate through Band.

01Read the room. Each agent pulls the live Band conversation as its primary context before it reasons.
02Challenge. When two agents disagree — an easement one finds and another misses — they resolve it in-thread.
03Delegate. Any agent can assign another a defined task, with intent and authority, derived from context.
04Recruit. The committee brings Python quantitative specialists into the room when the deal warrants it.
05Synthesize. A Deal Director composes a Red / Yellow / Green memo with conditions precedent.
Outcome

A deal memo a committee can defend, with the full reasoning attached, in minutes.

Decided at runtime

Which agents debate, who hands work to whom, and which specialists join are read from the deal, not fixed in code. The collaboration is emergent.

06
Band's role in coordination

What the committee relies on Band for.

Take any of these away and the committee stops working. We use the full Band surface, not just messaging.

Shared context

Agents read the room via mention-routed getContext before reasoning — primary context, not spoon-fed payloads.

Handoffs

Structured, mention-based posting passes the baton along the committee, each agent addressing the next.

Task state

Delegations are real Band tasks tracked processing → processed — accountable, not fire-and-forget.

Rich events

thought · tool_call · tool_result · error make each agent's reasoning visible and replayable.

Recruitment

Specialists are pulled into the live room mid-deal — cross-framework agents joining as first-class participants.

Distinct identity

Each specialist posts under its own Band identity (per-agent key) — genuinely separate agents.

Application of technology

The agents decide the next step.

lib/agents/schemas.ts
// each agent emits these from the deal context
export const ComplianceReportSchema = z.object({
  // …findings, risk_score…
  delegations: z.array(DelegationRequestSchema),
  requested_specialists: z.array(SpecialistRequestSchema),
});

const DelegationRequestSchema = z.object({
  to: z.enum(['archivist','regulatory','legal',
           'financial','synthesis']),
  intent: z.string(),     // what + why
  authority: z.string(),  // what may change
});
lib/orchestration/workflow.ts
// the agents decide who does what next
for (const d of delegations) {
  await delegate(dealId, roomId, d.from, d.to,
                 d.intent, d.authority, async () =>
    run(d.to, { ...ctx, delegation: d }));
  // → real Band task: processing → processed
}
What this enables

Delegations and specialist requests are values the agents produce, not control flow we wrote. A zoning conflict routes Legal to Financial; a contamination flag pulls in Environmental. Both come from the deal.
